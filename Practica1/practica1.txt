1. La diferencia más notoria al usar Haskell y Java, es el tipo de estructura que manejan
por n lado Haskell tiene una forma de código más conciso que Java, y Java tiene un estilo
llamado C que es conocido por usar bloques de código, delimitarse por medio de llaves y
terminar en punto y coma. Además difieren en que la concurencia de cada uno está dirigida
a diferentes enfoques, siendo en Java los multihilos y la capacidad de procesar código en
paralelo, es decir, que puede dividir su carga de trabajo utilizando múltiples procesadores,
mientras Haskell utiliza hilos ligeros. Y algo muy importante es que Java va orientado
principalmente a objetos que pueden contener datos y código para el desarrollo de aplicaciones
empresariales y móviles. Haskell utilza un lenguaje de programacón funcional que es más para
uso matemático.

2. El hecho de que Haskell no haya tenido un puesto tan reconocido en la industria del software
es por la baja familiaridaad de los programadores con su sintaxis, además de que el ecosistema
Java, así también como de otros, es más amplio que el de Haskell  dando como consecuencia que
hayan pocos programadores la utilicen y prefieran cosas modernas y más cómodas de utlizar.

3. Una analogía que pienso es que como se nos ha dicho, git es como un videojuego donde cada
vez que queremos guardar nuestro progreso se crea un checkpoint al que podemos volver si algo
hicimos mal, y github es cuando compartimos con otras personas la cuenta de nuestro juego que
lleva nuestro progreso para que puedan seguir jugando. No es tan buena analogía.
